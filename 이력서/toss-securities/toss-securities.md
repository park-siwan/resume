- **행동**: Figma 플러그인을 직접 개발하여 규격서 자동 추적 및 노션 블록 링크 자동 삽입 기능 구현. Claude Code + Figma Dev Mode MCP 통합으로 반복적인 UI 코드 생성 자동화
- **결과**: 디자인 변경사항 누락 방지, 규격서 기반 개발 정확도 향상

- **결과**: 컴포넌트 재사용성 및 유지보수성 대폭 향상, 레거시 코드 리팩토링 완료

- **행동**: Next.js 기반 SSR 렌더링 전략을 수립하고 반응형 웹 디자인으로 전면 리뉴얼을 설계·구현 (입사 6개월 만에 대규모 프로젝트 주도)
- **결과**: 다양한 디바이스에서의 사용자 경험 통합, 서비스 품질 향상

**백오피스 UI/UX 전면 개선**

- **행동**: 제품관리 페이지에 실시간 자동 검색, 멀티 필터링, 퀵서치 기능을 설계·구현. 뱃지 UI 추가와 필터 초기화로 전체보기 빠른 접근성 제공. 화면 새로고침 없이 실시간 상태 반영 시스템 구현
- **결과**: 관리자의 정보 탐색 효율 향상, 운영 업무 시간 절감

**데이터 시각화 UX 설계**

- **문제**: 투약 순응도 차트에서 60개 이상 데이터 포인트가 표시될 때 가로 스크롤이 필요하고 가독성이 떨어짐
- **행동**: Chart.js 기반 순응도/걸음수/투약횟수 그래프를 개발하면서, 대량 데이터 시 자동 생략 처리 알고리즘을 설계하여 스크롤 없이 핵심 정보를 전달하는 UX 구현
- **결과**: 의료진이 한눈에 환자 데이터를 파악할 수 있는 직관적 대시보드 완성

#### 성능 최적화 & 문제 해결

**페이지 로딩 속도 78.75% 개선**

- **문제**: 서비스 페이지 로딩이 8초로 사용자 이탈 위험이 높았음
- **행동**: 1차로 번들 분석 및 코드 스플리팅으로 8초 → 3초(62.5% 개선), 2차로 Vercel Edge 서버 최적화 및 서울 리전 고정으로 3초 → 1.7초(43.3% 추가 개선)
- **결과**: 최종 로딩 시간 **1.7초**, 총 **78.75% 개선**

**Next.js 메모리 누수 해결 및 502 장애 대응**

- **문제**: 프로덕션 환경에서 502 Bad Gateway 장애가 간헐적으로 발생
- **행동**: CPU 과부하의 근본 원인을 axios 메모리 누수로 파악하고 수정. HTTP/2 통신 구현으로 네트워크 효율성 추가 개선
- **결과**: **메모리 사용량 40% 감소**, 서비스 안정성 확보

**인프라 비용 최적화**

- **문제**: AWS ECS 기반 배포 환경의 운영 비용이 프로젝트 규모 대비 과도
- **행동**: Vercel로 배포 환경 전환을 제안하고 마이그레이션 주도
- **결과**: **연간 약 450만원 인프라 비용 절감**

#### 국제화(i18n) & 협업 시스템

**4개 국어 국제화 시스템 구축**

- next-i18next 기반 SSR/SSG 환경 국제화 설정, Accept-Language 헤더 기반 자동 언어 감지
- 한국어/중국어/일본어/영어 4개 국어 지원

- data-cy 속성 기반 테스트 타겟 표준화
