#### 1. 반응형 디자인, 웹 접근성, 웹 표준을 고려한 UI 개발

**인핸드케어웹 — 폰/태블릿/PC/네이티브 앱 웹뷰 전환경 반응형 설계**

- **문제**: 기존 서비스가 데스크톱 전용으로 설계되어 모바일 사용자의 접근성이 낮았고, 추후 네이티브 앱에서도 동일한 서비스를 제공해야 하는 요구사항이 있었음
- **행동**: Next.js 기반 반응형 웹 디자인으로 전면 리뉴얼을 설계·구현 (입사 6개월 만에 대규모 프로젝트 주도). 폰/태블릿/PC 3단계 브레이크포인트 대응은 물론, 이후 Android(Kotlin)/iOS(Swift) 네이티브 앱의 WebView에 동일 웹을 탑재하여 앱처럼 보이도록 구성
- **결과**: 하나의 웹 코드베이스로 모바일 웹, 데스크톱, 네이티브 앱(WebView) 3가지 환경을 동시에 서비스. 별도 네이티브 UI 개발 없이 웹뷰만으로 앱 사용자 경험 제공

**UI 트랜지션 패턴화 및 폼 접근성 설계**

- **문제**: 각 페이지/컴포넌트마다 애니메이션 속도와 이징이 제각각이어서 서비스 전체의 브랜드 경험이 일관되지 않았음. 또한 폼 에러 발생 시 사용자가 어디서 문제가 생겼는지 직관적으로 인지하기 어려웠음
- **행동**: `0.3s ease-in-out` 등 트랜지션 값을 중앙에서 정의하고 공통으로 전파하는 구조를 자발적으로 설계하여, 유저가 다음 행동을 예측할 수 있는 인터랙션의 연속성을 확보. 폼 에러 발생 시 해당 필드로 포커스 이동 및 스크롤 자동 이동하는 UX를 섬세하게 구성
- **결과**: 서비스 전반에 걸쳐 일관된 브랜드 경험과 예측 가능한 인터랙션 제공. 폼 입력 시 사용자가 에러 위치를 즉시 인지할 수 있는 UX 완성

**4개 국어 국제화 시스템 구축**

- next-i18next 기반 SSR/SSG 환경 국제화 설정, Accept-Language 헤더 자동 언어 감지
- 한국어/중국어/일본어/영어 4개 국어 지원
- 노션 데이터베이스와 번역 파일 자동 동기화 시스템 개발로 기획자/QA/개발자 협업 워크플로 구축

#### 2. 디자인 시스템 구축 및 유지보수

**Figma 시안 기반 컴포넌트 라이브러리 운영 (인핸드플러스)**

- **문제**: UI 컴포넌트가 파편화되어 디자인 일관성 유지가 어렵고, 디자이너와 개발자 간 스펙 불일치가 빈번했음
- **행동**: Figma Dev Mode에서 디자인 스펙을 확인하며 코드로 구현. Typography 컴포넌트를 props로 폰트사이즈/자간/행간/시맨틱태그를 제어할 수 있도록 설계하고, TypeScript 타입 강제로 잘못된 스타일 조합을 런타임 전에 차단
- **결과**: 디자인-개발 간 스타일 불일치 이슈 감소, 컴포넌트 재사용으로 UI 개발 속도 향상

**Atomic Design 기반 디자인 시스템 구축 (트리피누)**

- **문제**: 초기 스타트업에서 빠른 기능 개발과 UI 일관성을 동시에 달성해야 했음
- **행동**: Atoms/Molecules/Organisms 계층 구조로 재사용 가능한 컴포넌트 라이브러리를 처음부터 설계. Figma 시안을 참고하여 Typo 컴포넌트를 개발, props로 폰트사이즈/자간/행간/시맨틱태그/마진/패딩을 제어. **4-point grid system**을 적용해 픽셀 깨짐 방지. TypeScript로 props 타입을 강제하여 런타임 전 디버깅 가능
- **결과**: 디자이너가 정의한 디자인 스펙이 코드에 일관되게 반영되는 시스템 구축, 신규 페이지 개발 속도 향상

**아키텍처 선택의 트레이드오프 경험 (FSD → Atomic Design)**

- **문제**: 프로젝트 규모가 커지면서 컴포넌트 간 의존성이 복잡해지고, 새로운 기능 추가 시 사이드이펙트가 빈번하게 발생
- **행동**: Feature-Sliced Design 아키텍처를 학습하고 도입·적용. Entities, Features, Widgets, Pages 계층 구조를 설계하고 실제 프로젝트에서 운영
- **결과**: 모듈 간 참조 무결성은 확보했으나, 작은 UI 변경에도 수정해야 할 레이어가 많아져 빠른 실험·배포가 반복되는 환경에서 생산성 저하를 확인. 디자인 시스템과의 정합성이 높고 UI 컴포넌트 재사용을 극대화할 수 있는 Atomic Design으로 전환

#### 3. 공통 모듈 및 라이브러리 설계·유지보수

**Figma Plugin API를 활용한 디자인-개발 협업 자동화 도구 개발**

- **문제**: 디자인 규격서 변경 사항을 개발자가 수동으로 추적해야 했고, 규격서와 실제 구현 사이의 간극이 점차 벌어짐
- **행동**: Figma Plugin API를 활용해 규격서 자동 추적 및 노션 블록 링크 자동 삽입 기능을 직접 개발하여 팀에 제안
- **결과**: 최종 워크플로에는 채택되지 않았으나, 팀 내 협업 병목 지점을 파악하고 자동화 기술로 해결하려는 시도를 통해 디자이너의 작업 방식에 대한 깊은 이해를 얻음

**백오피스 UI/UX 전면 개선**

- **문제**: 관리자용 백오피스의 정보 탐색이 비효율적이고, 필터링/검색 UX가 직관적이지 않았음
- **행동**: 제품관리 페이지에 실시간 자동 검색, 멀티 필터링, 퀵서치 기능을 설계·구현. 뱃지 UI 추가와 필터 초기화로 전체보기 빠른 접근성 제공. 화면 새로고침 없이 실시간 상태 반영 시스템 구현
- **결과**: 관리자의 정보 탐색 효율 향상, 운영 업무 시간 절감

**데이터 시각화 UX 설계**

- **문제**: 투약 순응도 차트에서 60개 이상 데이터 포인트가 표시될 때 가로 스크롤이 필요하고 가독성이 떨어짐
- **행동**: Chart.js 기반 순응도/걸음수/투약횟수 그래프를 개발하면서, 대량 데이터 시 자동 생략 처리 알고리즘을 설계하여 스크롤 없이 핵심 정보를 전달하는 UX 구현
- **결과**: 의료진이 한눈에 환자 데이터를 파악할 수 있는 직관적 대시보드 완성

**반응형 웹 & 다국어 설계 (트리피누)**

- Material-UI Grid 시스템을 활용한 디바이스별 반응형 처리
- Sass/SCSS 아키텍처 설계 (base/constants/layouts/mixins/modules 구조)
- react-i18next 기반 한/영 다국어 지원, 브라우저 언어 자동 감지
- 재사용 가능한 Modal 컴포넌트, 이미지 preload, 미저장 방지 UX 등 공통 모듈 개발
